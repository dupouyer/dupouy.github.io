纹理

发表于 2018-05-14

属性

纹理是 3D 渲染中的一个重要工具，存储着2维数组，可用于多种用途。常见的类型有，diffuse，normal，lightmap。一般在 fragment shader 中使用。

在游戏开发中需要注意纹理一下两个方面：

尺寸

纹理尺寸必须为 2的冥常见尺寸有 256x256,1024x1024,2048x2048。
尺寸越大受带宽影响 GPU 加载也越慢。所以必须控制游戏在一帧的时间内，纹理的加载数量和大小。

格式

  • RGBA8888
    高清图，细节丰富，占用内存大
  • RGBA4444
    较清晰，细节有损失，比 32 位图内存小一半
  • ETC1
    GPU可直接读取，读取速度快，不支持 alpha 通道，清晰度低,占用内存小
  • ETC2
    GPU可直接读取，读取速度快，支持 alpha 通道，但是需要 OpenGl3.0 版本以上才支持
  • PVRC
    ios 下的GPU 格式，读取速度快，清晰度低，占用内存小

在游戏开发中，需要根据具体物体的远近，大小，关注度，来选取合适的纹理尺寸来应用。

性能优化

Batch

通过连续提交具有相同渲染状态的物体来减少GPU渲染状态的切换，提高渲染性能。

引擎上的优化

以 unity 为例，分为静态 batch 和动态 bacth

  • 静态 Batch
    在场景资源打包时，将具有相同材质物体的网格数据存入一段内存中。在实际渲染时，提交的是这段内存中的网格数据。会增大游戏包体大小。
  • 动态 Batch
    在运行时，把所有的物体变换到世界坐标后进行合并，实时使用 CPU 进行，对单次 batch 的物体数量有限制。

资源上的优化

使用 TextureAtlas，使更多物体具有相同的渲染状态。

引擎

  • 渲染管线:
    顶点 -> 几何 -> 片段 - > Ztest,Blend -> 栅格 -> 后处理
  • 物理碰撞的策略:
    • 精细化的选择不同的碰撞体
    • 合理的分层控制碰撞范围
    • 少用，或者不用连续碰撞检测
  • Batch:
    • TextureAtlas
    • 静态 batch，资源 build 阶段, 合并物体。
      优势：不消耗计算资源
      劣势：包体增大（生成资源文件用来描述合并后的物体）
    • 动态 batch 运行时，forward render ,CPU 合并 物体
      优势：灵活
      劣势：消耗计算资源切且，U3D 中对一次 batch 的定点数有限制 900 个，对于有透明物体的情况下，不能很好的执行合批。（透明物体有严格的渲染顺序控制）
    • 运行时合批: 调用 Mesh.CombineMeshes 方法合并物体
  • 纹理格式:

架构

  • MVC
    老生常谈的一个东西了，在业务逻辑的编写上要注意合理的拆分数据与操作逻辑，View 这一层一般都是由第三方的 UI 框架搭建，结构都很内聚。核心在于实现数据驱动。
  • UI
    • FariyGUI
    • UGUI
  • 面向对象
    • 多使用组合，避免使用继承。
    • 业务逻辑上继承的越多，维护越困难。
    • 多使用方法操作数据
    • 业务数据结构要足够元,才能更好的应对迭代变化, 变化的部分放入方法中。

适配

  • shader

性能优化

代码(CPU):

  • 减少业务层的复杂逻辑，运算量大的使用 C# 执行，避免使用 Lua
